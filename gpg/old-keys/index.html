Old GPG Keys

I've gone through a few iterations of keys in April 2011 while working out a policy I should follow in dealing with keys.

Short history of keys:

  • First iteration used a web of multiple independent keys
  • Second iteration used a single master key of huge size (8192-RSA)
  • Final iteration in April 2011: single master key (3072-RSA)

Archived history details

I've switched from many keys with chain of trust (listed below and revoked) to one key with many subkeys.

  • Primary for important signatures and certification.
  • 2 subkeys primarily for email / file encryption
  • Signing subkey for code signing/etc

The primary key is only accessible from a USB drive that is accessed in a very protected state. The other keys are relatively short-lived and "less" protected.
